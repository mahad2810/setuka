## The Problem

Adventure tourism is expanding at **19% CAGR**, yet safety infrastructure in mountain corridors is stagnant — creating a deadly **Triple Threat**:

| Gap | Reality |
|-----|---------|
| **Connectivity Void** | 40% of key corridors are digital dark zones. Satellite alternatives (e.g. Garmin) are **legally banned** in Indian border regions. |
| **Information Vacuum** | Authorities operate in silos — zero real-time crowd or route data, purely reactive disaster management. |
| **Response Lag** | Without coordinates or health vitals, emergency response is delayed 15–30 min, turning survivable injuries fatal. |

---

## The Solution

Setuka is a **three-layer IoT + AI ecosystem** that wraps every phase of a tourist's journey:

### Layer 1 — Tourist App (PWA)
A Next.js Progressive Web App (installable on Android/iOS) that gives each traveller:
- **Live Safety Score (0–100):** ML-computed score for their exact location using crime, road-condition, and accident-risk indexes.
- **Itinerary Planner:** Routes ranked by safety score with AI narrative generated by Groq LLM.
- **Blockchain Digital ID:** Tamper-proof ID on Polygon network with medical + insurance info — accessible offline via QR code, no internet needed.
- **Dynamic Geofencing:** Real-time warnings when the tourist enters a flagged high-risk zone.
- **Emergency SOS:** One-tap alert with live coordinates pushed to the authority dashboard.

### Layer 2 — IoT LoRa Wearable (Hardware)
A custom-built wearable (9-phase hardware build) with:
- **Sensors:** LoRa SX1262, GPS (Quectel EC200U), SpO₂ / pulse oximeter, GPS, GNSS, OLED display.
- **Triple-Mode SOS Transmission:**
  1. Direct Cloud — when internet is available.
  2. SMS — via GSM in partial-signal areas.
  3. LoRa P2P Mesh — zero-connectivity zones; signal hops device-to-device until it reaches a receiver.
- **Auto-SOS:** Triggers automatically on critical health-vital anomalies.

### Layer 3 — Authority Dashboard (Streamlit)
A command-center for tourism police and district authorities:
- **Live Tourist Map:** Real-time cluster map of all active tourists.
- **Crowd Safety Scoring:** DBSCAN clustering assigns risk levels to GPS observations.
- **Anomaly & Inactivity Detection:** Isolation Forest flags route deviations and unusual stops.
- **AI Recommendations:** LLM-powered (Groq + Gemini via LangChain) suggestions for crowd redistribution and police reallocation to high-risk zones.

## Machine Learning Details

### Safety Score Model
Two region-specific **XGBoost regression** models (Kolkata & Darjeeling) trained on:
- Crime level index
- Road condition index
- Accident risk index

Output is normalised to **0–100**; a Groq LLM then generates a human-readable safety narrative.

### Crowd Analysis
**DBSCAN** clustering on live GPS observations from the police patrol dataset assigns risk levels (Low / Medium / High / Critical) to geographic clusters.

### Anomaly Detection
**Isolation Forest** pipeline in `isolation_module/` detects:
- Route deviations from planned itinerary
- Unusual inactivity (potential distress)
- Statistical outliers in movement patterns
